## P0: Must-Have at Launch

### 1. Storage Abuse Protection

**The #1 financial risk.** A scripted bot can upload thousands of files and explode your S3 bill.

**Controls:**
- Hard upload cap per event tier (150 free, unlimited paid)
- Atomic counter in DynamoDB (`uploadCount`) — checked before issuing presigned URL
- Max file size enforced in presigned POST conditions:
  - Free: 2MB images
  - Paid: 5MB images, 30MB video, 10MB audio
- Per-session upload rate: max 10 uploads/minute
- Per-IP rate limiting at API Gateway level
- Video disabled entirely on free tier
- S3 lifecycle rules auto-delete free events after 30 days

### 2. Malicious File Upload Prevention

**Risks:** Executable files disguised as images, XSS via SVG, malware hosting.

**Controls:**
- **Whitelist MIME types** in presigned URL:
  - Images: `image/jpeg`, `image/png`, `image/webp`, `image/heic`
  - Video (paid only): `video/mp4`, `video/quicktime`
  - Audio (paid only): `audio/mpeg`, `audio/mp4`, `audio/wav`
- **Block SVG entirely** (XSS vector)
- **Validate Content-Type header** matches file extension
- **S3 trigger Lambda** validates file magic bytes after upload:
  - JPEG: `FF D8 FF`
  - PNG: `89 50 4E 47`
  - MP4: `66 74 79 70` (ftyp box)
  - If invalid → delete immediately + decrement counter
- **CloudFront response headers:**
  - `Content-Disposition: attachment` for downloads
  - `X-Content-Type-Options: nosniff`
  - `Content-Security-Policy: default-src 'none'`
- **No server-side rendering of uploaded content** — thumbnails generated by Lambda

### 3. Private Media (No Public S3)

**Controls:**
- **S3 Block Public Access:** ALL four settings enabled
- **Bucket policy:** Deny all public access explicitly
- **CloudFront Origin Access Control (OAC):** Only CloudFront can read from S3
- **Signed URLs for reads:** All media URLs are CloudFront signed URLs with 1-hour expiry
- **Random UUIDs for file names** — no guessable paths
- **No directory listing** — CloudFront default root object only for SPA

### 4. Unauthorized Event Access

**Controls:**
- **Host password:** bcrypt hashed (cost factor 12)
- **Guest password:** compared server-side only
- **Rate limiting on auth endpoint:** 10 attempts per IP per 5 minutes
- **Exponential backoff** returned in 429 response
- **Event auto-lock** after `endDate` + grace period
- **Session tokens (JWT):**
  - Short-lived (guest: event duration + 2h, host: 24h)
  - Signed with HS256 + secret from Secrets Manager
  - Include `eventId` + `role` claims
  - Validated on every authenticated request

## P1: Should-Have Before Public Launch

### 5. Presigned URL Hardening

- Expiry: **3 minutes** (not longer)
- **Content-Length-Range** condition: min 1 byte, max per tier limit
- **Content-Type** condition: must match requested type
- **S3 key** scoped to exact path (Lambda-generated, not user-controlled)
- **One URL = one file** — key includes unique mediaId
- Track issued presigned URLs in DynamoDB (optional, for audit)

### 6. DDoS / Traffic Spike Protection

- **AWS WAF** on CloudFront:
  - Rate-based rule: 100 requests/5min per IP
  - AWS managed bot control rule group
  - IP reputation rule group
  - Geographic restriction (optional — start with GT + US only)
- **API Gateway throttling:**
  - Default: 1000 requests/sec burst, 500 sustained
  - Per-route throttling for sensitive endpoints (auth, upload-url)
- **CloudFront caching:**
  - Cache static assets aggressively (1 year, immutable)
  - Cache media thumbnails (24h)
  - Don't cache API responses
- **Lambda reserved concurrency** — prevent runaway invocations from killing budget:
  - processUpload: max 10 concurrent
  - Other functions: max 50 concurrent total

## P2: Nice-to-Have (Post-Launch)

### 7. Content Moderation (Premium Tier — Rekognition)

- **Amazon Rekognition** DetectModerationLabels (**Premium tier only**):
  - Run on S3 trigger after upload (processUpload Lambda)
  - Auto-detect: nudity, violence, drugs, hate symbols, disturbing content
  - Flagged content auto-hidden from gallery with "Under review" placeholder
  - Host reviews flagged content in moderation dashboard → approve or reject
  - Scans thumbnails (800px) not originals — faster and cheaper
  - Cost: ~$0.001/image, ~$0.10/minute video
  - Graceful degradation: if Rekognition fails, image defaults to visible
- **Host moderation controls (all tiers):**
  - Delete any media (all tiers)
  - Ban session (all tiers)
  - Approve/reject flagged content (Premium)
  - Moderation dashboard with stats (Premium)
- **Comment sanitization (all tiers):**
  - Strip all HTML tags
  - Profanity filter (basic word list, Premium)
  - Max 500 characters
- See `docs/12-CONTENT-MODERATION.md` for full details

### 8. Payment Fraud Prevention

- Limit free events per IP: 3 per day
- Require email for event creation (soft verification)
- CAPTCHA on event creation (hCaptcha — free tier available)
- Track event creation velocity per email
- Recurrente handles card fraud on their side

### 9. Host Account Security

- bcrypt with cost 12
- Optional email OTP for host login (future)
- Host session invalidation on password change
- All host actions logged in DynamoDB

## P3: Future

- Auto-generated nicknames (`Guest_1`, `Guest_2`, etc.)
- Optional nickname entry (validated, no HTML)
- Tied to session token — can't impersonate
- IP hash stored for abuse tracking (not displayed)

## IAM Principle of Least Privilege

Each Lambda function gets a scoped IAM role:

| Function | Permissions |
|---|---|
| createEvent | `dynamodb:PutItem` on EventAlbum table |
| authEvent | `dynamodb:GetItem`, `dynamodb:PutItem` (sessions) |
| getUploadUrl | `dynamodb:GetItem`, `s3:PutObject` (scoped to events/* prefix) |
| processUpload | `s3:GetObject`, `s3:PutObject` (thumbs), `s3:DeleteObject`, `dynamodb:UpdateItem` |
| listMedia | `dynamodb:Query` |
| addReaction | `dynamodb:PutItem`, `dynamodb:UpdateItem` |
| addComment | `dynamodb:PutItem`, `dynamodb:UpdateItem` |
| createCheckout | `dynamodb:GetItem`, `dynamodb:UpdateItem` (no S3) |
| handleWebhook | `dynamodb:UpdateItem` |

**No Lambda function has `s3:*` or `dynamodb:*`.** Always scoped to specific actions + resources.
