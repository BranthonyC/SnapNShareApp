# CloudFront Configuration

## Distributions

### 1. Media CDN (`cdn.eventalbum.app`)

- **Purpose:** Serve media files (images, thumbnails, video, audio) via signed URLs
- **Origin:** S3 `eventalbum-media-{env}` via OAC (Origin Access Control)
- **No public access** — all URLs must be signed
- **Price class:** PriceClass_100 (US, Canada, Europe) — cheapest, covers our markets
- **HTTP version:** HTTP/2 and HTTP/3 enabled
- **IPv6:** Enabled

#### Origin Access Control (OAC)

- **Type:** S3
- **Signing behavior:** Always sign
- **Signing protocol:** SigV4
- **Origin:** S3 bucket domain (not S3 website endpoint)
- **Origin shield:** Disabled (not needed for single-region origin)

```bash
# Create OAC
aws cloudfront create-origin-access-control \
  --origin-access-control-config '{
    "Name": "EventAlbum-Media-OAC",
    "Description": "OAC for EventAlbum media bucket",
    "SigningProtocol": "sigv4",
    "SigningBehavior": "always",
    "OriginAccessControlOriginType": "s3"
  }' \
  --profile codersatelier
```

#### Behaviors

| Path Pattern | Allowed Methods | Cache Policy | Signed | Compress |
|---|---|---|---|---|
| `/events/*/thumbs/*` | GET, HEAD | 30 days TTL | Yes (CloudFront signed URL) | Yes |
| `/events/*/originals/*` | GET, HEAD | 7 days TTL | Yes | No (already compressed images) |
| `/exports/*` | GET, HEAD | 1 day TTL | Yes | No (ZIP files) |
| Default (`*`) | GET, HEAD | 1 day TTL | Yes | Yes |

#### Signed URLs

- **Key type:** CloudFront key group (RSA 2048-bit) — NOT root account key pairs
- **Expiry:** 1 hour for image/thumbnail reads, 4 hours for video, 24 hours for ZIP downloads
- **Generated by:** Lambda on each `listMedia` / `getEvent` / `downloadZip` response
- **Policy:** Custom policy restricting to specific S3 key path prefix

##### Key Group Setup

```bash
# Generate RSA key pair (do this locally, NOT on EC2)
openssl genrsa -out cf-private-key.pem 2048
openssl rsa -in cf-private-key.pem -pubout -out cf-public-key.pem

# Upload public key to CloudFront
aws cloudfront create-public-key \
  --public-key-config '{
    "CallerReference": "eventalbum-cf-key-2024",
    "Name": "EventAlbum-SigningKey",
    "EncodedKey": "'$(cat cf-public-key.pem)'",
    "Comment": "CloudFront signing key for EventAlbum media"
  }' \
  --profile codersatelier

# Create key group using the public key ID returned above
aws cloudfront create-key-group \
  --key-group-config '{
    "Name": "EventAlbum-KeyGroup",
    "Items": ["<PUBLIC_KEY_ID>"],
    "Comment": "Key group for EventAlbum signed URLs"
  }' \
  --profile codersatelier

# Store private key in SSM Parameter Store (SecureString)
aws ssm put-parameter \
  --name "/eventalbum/dev/cloudfront-private-key" \
  --type SecureString \
  --value file://cf-private-key.pem \
  --profile codersatelier

# Store key pair ID in SSM
aws ssm put-parameter \
  --name "/eventalbum/dev/cloudfront-key-pair-id" \
  --type String \
  --value "<PUBLIC_KEY_ID>" \
  --profile codersatelier

# SECURELY DELETE the local private key after storing in SSM
rm cf-private-key.pem
```

##### Signed URL Generation (Lambda)

```typescript
import { getSignedUrl } from '@aws-sdk/cloudfront-signer';

const signedUrl = getSignedUrl({

  keyPairId: process.env.CLOUDFRONT_KEY_PAIR_ID,
  privateKey: cloudFrontPrivateKey,  // loaded from SSM at cold start

```

#### Cache Policy: `EventAlbum-MediaCache`

```bash
aws cloudfront create-cache-policy \
  --cache-policy-config '{
    "Name": "EventAlbum-MediaCache",
    "DefaultTTL": 86400,
    "MinTTL": 1,
    "MaxTTL": 2592000,
    "ParametersInCacheKeyAndForwardedToOrigin": {
      "EnableAcceptEncodingGzip": true,
      "EnableAcceptEncodingBrotli": true,
      "HeadersConfig": {"HeaderBehavior": "none"},
      "CookiesConfig": {"CookieBehavior": "none"},
      "QueryStringsConfig": {"QueryStringBehavior": "none"}
    },
    "Comment": "Cache media by path only, 1 day default, 30 day max"
  }' \
  --profile codersatelier
```

- **Min TTL:** 1 second
- **Default TTL:** 86,400 (1 day)
- **Max TTL:** 2,592,000 (30 days)
- **Cache key:** Path only (no query strings, no headers, no cookies)
- **Compression:** Gzip + Brotli enabled

#### Error Pages (Media CDN)

No custom error pages on media CDN. If a signed URL is invalid or expired, CloudFront returns 403 natively and the client handles it.

### 2. Frontend CDN (`eventalbum.app` / `www.eventalbum.app`)

- **Origin:** S3 `eventalbum-frontend-{env}` via OAC
- **Default root object:** `index.html`
- **Price class:** PriceClass_100
- **HTTP version:** HTTP/2 and HTTP/3 enabled
- **IPv6:** Enabled

#### OAC Setup

Same approach as media CDN — create a separate OAC for the frontend bucket.

```bash
aws cloudfront create-origin-access-control \
  --origin-access-control-config '{
    "Name": "EventAlbum-Frontend-OAC",
    "Description": "OAC for EventAlbum frontend bucket",
    "SigningProtocol": "sigv4",
    "SigningBehavior": "always",
    "OriginAccessControlOriginType": "s3"
  }' \
  --profile codersatelier
```

#### Behaviors

| Path Pattern | Allowed Methods | Cache TTL | Notes |
|---|---|---|---|
| `/assets/*` | GET, HEAD | 1 year (31536000s) | Vite hashed filenames = immutable cache |
| `*.html` | GET, HEAD | 0 (no-cache) | Always fetch latest HTML |
| Default (`*`) | GET, HEAD | 1 day (86400s) | Fallback for other static assets |

#### Cache Policies per Behavior

**`/assets/*` — Immutable Assets**
```
Min TTL: 31536000
Default TTL: 31536000
Max TTL: 31536000
```
Vite generates hashed filenames (`index-a1b2c3.js`), so these are safe to cache forever.

**`*.html` — No Cache**
```
Min TTL: 0
Default TTL: 0
Max TTL: 0
```
HTML files must always be fresh to pick up new deployments.

**Default — Standard Cache**
```
Min TTL: 0
Default TTL: 86400
Max TTL: 86400
```

#### Custom Error Responses (SPA Routing)

| HTTP Error Code | Response Path | Response Code | TTL |
|---|---|---|---|
| 403 | `/index.html` | 200 | 0 seconds |
| 404 | `/index.html` | 200 | 0 seconds |

This is essential for client-side routing — any path that doesn't match an S3 object returns `index.html` with 200, letting React Router handle the route.

## SSL/TLS Certificate

### ACM Certificate (Must Be in us-east-1 for CloudFront)

```bash
# Request wildcard + apex certificate
aws acm request-certificate \
  --domain-name eventalbum.app \
  --subject-alternative-names "*.eventalbum.app" \
  --validation-method DNS \
  --region us-east-1 \
  --profile codersatelier

# After ACM provides DNS validation records, add them to Route 53
# ACM will auto-validate once DNS propagates (usually 5-30 minutes)
```

**Certificate covers:**
- `eventalbum.app` (frontend prod)
- `*.eventalbum.app` (cdn, api, staging, www — all subdomains)

|---|---|
| Minimum protocol | TLSv1.2_2021 |
| SSL support method | SNI (free, no dedicated IP needed) |

## Cache Invalidation

### After Frontend Deployment

```bash
# Invalidate HTML and root files (hashed assets don't need invalidation)
aws cloudfront create-invalidation \
  --distribution-id <FRONTEND_DISTRIBUTION_ID> \
  --paths "/index.html" "/manifest.json" "/robots.txt" \
  --profile codersatelier
```

**Cost:** First 1,000 invalidation paths/month free. Each path after: $0.005.

### After Media Deletion

When an event host deletes media, the Lambda should invalidate the specific CloudFront path:

```typescript
import { CloudFrontClient, CreateInvalidationCommand } from '@aws-sdk/client-cloudfront';

const cf = new CloudFrontClient({ region: 'us-east-1' });

await cf.send(new CreateInvalidationCommand({
  DistributionId: process.env.CLOUDFRONT_DISTRIBUTION_ID,
  InvalidationBatch: {
    CallerReference: `${mediaId}-${Date.now()}`,
    Paths: {
      Quantity: 2,
      Items: [
        `/events/${eventId}/originals/${mediaId}.*`,
        `/events/${eventId}/thumbs/${mediaId}.*`,
      ],
    },
  },
}));
```

## Monitoring

### CloudFront Metrics (Free)

| Metric | Alarm Threshold | Notes |
|---|---|---|
| `Requests` | > 10,000/hour | Unusual traffic spike |
| `4xxErrorRate` | > 10% | Client errors (expired signed URLs) |
| `5xxErrorRate` | > 5% | Origin errors |
| `BytesDownloaded` | > 10 GB/day | Monitor bandwidth costs |

```bash

```

Access logs are optional — enable only if you need detailed analytics. CloudWatch metrics are sufficient for monitoring.

## Security Checklist

- [x] OAC configured (not legacy OAI)
- [x] S3 bucket policy restricts to CloudFront distribution ARN
- [x] All media served via signed URLs (custom policy, not canned)
- [x] Signed URL expiry: 1 hour (images), 4 hours (video), 24 hours (ZIP)
- [x] RSA private key stored in SSM SecureString (not in Lambda env vars)
- [x] TLS 1.2 minimum enforced
- [x] Security headers applied (HSTS, nosniff, DENY framing)
- [x] No public access to S3 origins
- [x] Price class limited to reduce attack surface from unexpected regions
- [x] HTTP/3 enabled for improved performance
