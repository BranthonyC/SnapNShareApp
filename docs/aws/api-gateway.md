# API Gateway Configuration

## Type: HTTP API (not REST API)

**Why HTTP API over REST API?**

| Feature | HTTP API | REST API |
|---|---|---|
| Cost | $1.00/million requests | $3.50/million requests |
| Latency | Lower (~10ms routing) | Higher (~29ms routing) |
| JWT authorizer | Native, built-in | Custom Lambda authorizer required |
| CORS | Automatic configuration | Manual OPTIONS handling |
| Payload size | 10 MB | 10 MB |
| WebSocket | No | No (separate WebSocket API) |
| Usage plans/API keys | No | Yes |
| Request validation | No (validate in Lambda) | Yes (JSON Schema) |
| WAF integration | Yes (via stage ARN) | Yes |
| Caching | No built-in | Yes (0.5-237 GB) |

**Decision:** HTTP API. 70% cheaper, native JWT, lower latency. We don't need caching (DynamoDB is fast), usage plans, or request validation at the gateway level.

## CORS Configuration

```bash
aws apigatewayv2 update-api \
  --api-id <API_ID> \
  --cors-configuration '{
    "AllowOrigins": ["https://eventalbum.app","https://staging.eventalbum.app","http://localhost:5173"],
    "AllowMethods": ["GET","POST","PATCH","DELETE","OPTIONS"],
    "AllowHeaders": ["Authorization","Content-Type","X-Request-Id"],
    "AllowCredentials": false,
    "MaxAge": 86400
  }' \
  --profile codersatelier
```

| Setting | Value | Notes |
|---|---|---|
| AllowOrigins | `https://eventalbum.app`, `https://staging.eventalbum.app`, `http://localhost:5173` | localhost only in dev |
| AllowMethods | GET, POST, PATCH, DELETE, OPTIONS | All methods our API uses |
| AllowHeaders | Authorization, Content-Type, X-Request-Id | JWT token, JSON body, request tracing |
| AllowCredentials | false | Not using cookies for auth |
| MaxAge | 86400 (24 hours) | Browser caches preflight response |

**Note:** HTTP API handles OPTIONS preflight automatically. No need for explicit OPTIONS routes.

## JWT Authorizer

```bash
aws apigatewayv2 create-authorizer \
  --api-id <API_ID> \
  --name "EventAlbum-JWT" \
  --authorizer-type JWT \
  --identity-source '$request.header.Authorization' \
  --jwt-configuration '{
    "Issuer": "https://api.eventalbum.app",
    "Audience": ["eventalbum-app"]
  }' \
  --profile codersatelier
```

| Setting | Value |
|---|---|
| Type | JWT |
| Identity source | `$request.header.Authorization` |
| Issuer | `https://api.eventalbum.app` |
| Audience | `eventalbum-app` |

**JWT Payload (custom, generated by our authEvent Lambda):**

```json
{
  "sub": "evt_abc123",
  "aud": "eventalbum-app",
  "iss": "https://api.eventalbum.app",
  "iat": 1700000000,
  "exp": 1700086400,
  "role": "host | guest",
  "eventId": "evt_abc123",
  "guestId": "guest_xyz"
}
```

**Authorization logic:** API Gateway validates JWT signature, expiry, issuer, and audience. Role-based access (host vs guest) is checked inside each Lambda function.

## Throttling

### Account-Level Defaults

| Setting | Value |
|---|---|
| Burst limit | 100 requests |
| Rate limit | 50 requests/second |

```bash
aws apigatewayv2 update-stage \
  --api-id <API_ID> \
  --stage-name '$default' \
  --default-route-settings '{
    "ThrottlingBurstLimit": 100,
    "ThrottlingRateLimit": 50
  }' \
  --profile codersatelier
```

**Per-route throttling is NOT available on HTTP APIs.** For rate-limiting specific endpoints (e.g., createEvent, sendOtp), implement in-Lambda rate limiting using DynamoDB counters.

### Lambda-Level Rate Limiting Strategy

| Endpoint | Limit | Window | Key |
|---|---|---|---|
| POST /events | 5 events | 1 hour | IP address |
| POST /events/{eventId}/auth | 10 attempts | 15 minutes | IP + eventId |
| POST /events/{eventId}/otp/send | 3 sends | 5 minutes | eventId + phone/email |
| POST /events/{eventId}/upload-url | 100 URLs | 1 hour | eventId + guestId |
| POST /webhooks/recurrente | No limit | — | Verified by Recurrente |

### Route Creation CLI Example

```bash
# Create integration (one per Lambda function)
aws apigatewayv2 create-integration \
  --api-id <API_ID> \
  --integration-type AWS_PROXY \
  --integration-uri arn:aws:lambda:us-east-1:<ACCOUNT_ID>:function:EventAlbum-createEvent-dev \
  --payload-format-version 2.0 \
  --profile codersatelier

# Create route (public, no auth)
aws apigatewayv2 create-route \
  --api-id <API_ID> \
  --route-key "POST /events" \
  --target "integrations/<INTEGRATION_ID>" \
  --profile codersatelier

# Create route (authenticated)
aws apigatewayv2 create-route \
  --api-id <API_ID> \
  --route-key "GET /events/{eventId}" \
  --target "integrations/<INTEGRATION_ID>" \
  --authorization-type JWT \
  --authorizer-id <AUTHORIZER_ID> \
  --profile codersatelier
```

**Note:** In practice, all routes and integrations are defined in the SAM template (`template.yaml`), not created manually.

## Custom Domain

```bash
# Create custom domain mapping
aws apigatewayv2 create-domain-name \
  --domain-name api.eventalbum.app \
  --domain-name-configurations '[{
    "CertificateArn": "arn:aws:acm:us-east-1:<ACCOUNT_ID>:certificate/<CERT_ID>",
    "EndpointType": "REGIONAL",
    "SecurityPolicy": "TLS_1_2"
  }]' \
  --profile codersatelier

# Map API to domain
aws apigatewayv2 create-api-mapping \
  --api-id <API_ID> \
  --domain-name api.eventalbum.app \
  --stage '$default' \
  --profile codersatelier

# Add Route 53 A record (alias)
# Target: the API Gateway domain name returned by create-domain-name
# e.g., d-xxxxx.execute-api.us-east-1.amazonaws.com
```

| Setting | Value |
|---|---|
| Domain | `api.eventalbum.app` |
| Certificate | ACM `*.eventalbum.app` (us-east-1 for regional endpoint) |
| Endpoint type | Regional |
| Security policy | TLS 1.2 |
| Base path | `/` (no prefix) |

## Access Logging

```bash
# Create CloudWatch log group
aws logs create-log-group \
  --log-group-name /aws/apigateway/EventAlbum-dev \
  --profile codersatelier

aws logs put-retention-policy \
  --log-group-name /aws/apigateway/EventAlbum-dev \
  --retention-in-days 30 \
  --profile codersatelier

# Enable access logging on stage
aws apigatewayv2 update-stage \
  --api-id <API_ID> \
  --stage-name '$default' \
  --access-log-settings '{
    "DestinationArn": "arn:aws:logs:us-east-1:<ACCOUNT_ID>:log-group:/aws/apigateway/EventAlbum-dev",
```

## Error Responses

HTTP API returns standardized error responses:

| Status | Body | When |
|---|---|---|
| 401 | `{"message": "Unauthorized"}` | Missing/invalid JWT |
| 403 | `{"message": "Forbidden"}` | JWT valid but scope insufficient |
| 404 | `{"message": "Not Found"}` | No matching route |
| 429 | `{"message": "Too Many Requests"}` | Throttle limit exceeded |

Lambda functions return custom error responses:

```json
{
  "statusCode": 400,
  "body": "{\"error\":\"VALIDATION_ERROR\",\"message\":\"Event title is required\"}"
}
```
