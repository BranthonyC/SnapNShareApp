# S3 Configuration

## Buckets

### 1. Media Bucket (`eventalbum-media-{env}`)

- **Purpose:** Store event uploads (images, video, audio), thumbnails, covers, ZIP exports
- **Region:** us-east-1
- **Public access:** BLOCKED (all 4 settings enabled)
- **Encryption:** SSE-S3 (AES-256) — free, no KMS costs
- **Versioning:** Disabled (saves cost, we don't need version history)
- **Object ownership:** BucketOwnerEnforced (disables ACLs entirely)

#### Key Prefixes

```
events/{eventId}/originals/{mediaId}.{ext}     # Full-size uploads
events/{eventId}/thumbs/{mediaId}.webp          # Thumbnails (generated by processUpload Lambda)
events/{eventId}/cover.webp                      # Event cover image
events/{eventId}/qr.png                          # Generated QR code
exports/{eventId}/{timestamp}.zip                # ZIP download bundles
```

#### CORS Configuration

```json
[
  {
    "AllowedHeaders": ["Content-Type"],
    "AllowedMethods": ["PUT"],
    "AllowedOrigins": [
      "https://eventalbum.app",
      "https://staging.eventalbum.app",
      "http://localhost:5173"
    ],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3600
  }
]
```

- Only PUT (presigned uploads) — no GET/DELETE from browser
- GET served via CloudFront signed URLs
- `ETag` exposed so the client can confirm upload integrity

#### Lifecycle Rules

| Rule | Prefix/Tag | Transition | Action |
|---|---|---|---|
| Free tier cleanup | Tag: `tier=free` | — | Delete after 15 days (matches DynamoDB event TTL) |
| Paid tier Glacier | Tag: `tier=paid` | STANDARD -> GLACIER after 90 days | Delete after 180 days |
| Premium tier deep archive | Tag: `tier=premium` | STANDARD -> GLACIER after 90 days, GLACIER -> DEEP_ARCHIVE after 180 days | Delete after 730 days (2 years) |
| Incomplete multipart | — | — | AbortIncompleteMultipartUpload after 1 day |
| ZIP exports cleanup | Prefix: `exports/` | — | Delete after 7 days |

**Note:** Per-event lifecycle is managed via S3 object tags. Each event's objects get tagged with `tier=free|paid|premium` at upload time. Lifecycle rules filter by tag.

#### S3 Object Tags

```
tier: free | paid | premium
eventId: evt_xxx
uploadedBy: guest_{guestId} | host
```

#### Bucket Policy

Only allow access from CloudFront OAC and Lambda execution roles. No direct public access ever.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowCloudFrontOAC",
      "Effect": "Allow",
      "Principal": {
        "Service": "cloudfront.amazonaws.com"
      },
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::eventalbum-media-{env}/*",
      "Condition": {
        "StringEquals": {
          "AWS:SourceArn": "arn:aws:cloudfront::{account-id}:distribution/{distribution-id}"
        }
      }
    },
    {
      "Sid": "DenyUnencryptedUploads",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:PutObject",
      "Resource": "arn:aws:s3:::eventalbum-media-{env}/*",
      "Condition": {
        "StringNotEquals": {
          "s3:x-amz-server-side-encryption": "AES256"
        }
      }
    }
  ]
}
```

#### CLI Setup

```bash
# Create bucket
aws s3api create-bucket \
  --bucket eventalbum-media-dev \
  --region us-east-1 \
  --profile codersatelier

# Block all public access
aws s3api put-public-access-block \
  --bucket eventalbum-media-dev \
  --public-access-block-configuration \
    BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true \
  --profile codersatelier

# Enable SSE-S3 encryption
aws s3api put-bucket-encryption \
  --bucket eventalbum-media-dev \
  --server-side-encryption-configuration \
    '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"AES256"},"BucketKeyEnabled":true}]}' \
  --profile codersatelier

# Set object ownership to BucketOwnerEnforced (disable ACLs)
aws s3api put-bucket-ownership-controls \
  --bucket eventalbum-media-dev \
  --ownership-controls '{"Rules":[{"ObjectOwnership":"BucketOwnerEnforced"}]}' \
  --profile codersatelier

# Apply CORS configuration
aws s3api put-bucket-cors \
  --bucket eventalbum-media-dev \
  --cors-configuration '{
    "CORSRules": [
      {
        "AllowedHeaders": ["Content-Type"],
        "AllowedMethods": ["PUT"],
        "AllowedOrigins": ["https://eventalbum.app","https://staging.eventalbum.app","http://localhost:5173"],
        "ExposeHeaders": ["ETag"],
        "MaxAgeSeconds": 3600
      }
    ]
  }' \
  --profile codersatelier

# Apply lifecycle rules
aws s3api put-bucket-lifecycle-configuration \
  --bucket eventalbum-media-dev \
  --lifecycle-configuration '{
    "Rules": [
      {
        "ID": "FreeTierCleanup",
        "Status": "Enabled",
        "Filter": {"Tag": {"Key": "tier", "Value": "free"}},
        "Expiration": {"Days": 15}
      },
      {
        "ID": "PaidTierGlacier",
        "Status": "Enabled",
        "Filter": {"Tag": {"Key": "tier", "Value": "paid"}},
        "Transitions": [
          {"Days": 90, "StorageClass": "GLACIER"}
        ],
        "Expiration": {"Days": 180}
      },
      {
        "ID": "PremiumTierDeepArchive",
        "Status": "Enabled",
        "Filter": {"Tag": {"Key": "tier", "Value": "premium"}},
        "Transitions": [
          {"Days": 90, "StorageClass": "GLACIER"},
          {"Days": 180, "StorageClass": "DEEP_ARCHIVE"}
        ],
        "Expiration": {"Days": 730}
      },
      {
        "ID": "AbortIncompleteMultipart",
        "Status": "Enabled",
        "Filter": {},
        "AbortIncompleteMultipartUpload": {"DaysAfterInitiation": 1}
      },
      {
        "ID": "ZipExportsCleanup",
        "Status": "Enabled",
        "Filter": {"Prefix": "exports/"},
        "Expiration": {"Days": 7}
      }
    ]
  }' \
  --profile codersatelier

# Apply bucket policy
aws s3api put-bucket-policy \
  --bucket eventalbum-media-dev \
  --policy file://s3-media-bucket-policy.json \
  --profile codersatelier
```

#### Presigned URL Generation (Lambda)

```typescript
import { S3Client, PutObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

const s3 = new S3Client({ region: 'us-east-1' });

const command = new PutObjectCommand({
  Bucket: process.env.MEDIA_BUCKET,
  Key: `events/${eventId}/originals/${mediaId}.${ext}`,
  ContentType: contentType,
  ServerSideEncryption: 'AES256',
  Tagging: `tier=${tier}&eventId=${eventId}`,
  Metadata: {
    'uploaded-by': guestId,
    'event-id': eventId,
  },
});

const presignedUrl = await getSignedUrl(s3, command, {
  expiresIn: 300,  // 5 minutes
});
```

- Presigned URL expiry: **5 minutes** (short — upload immediately after requesting)
- Content-Type restricted at presigned time (prevents uploading non-media files)
- Server-side encryption enforced in the presigned command
- Tags applied at upload time for lifecycle management

#### Size Limits (Enforced via Presigned URL Conditions)

| Tier | Max File Size | Max Total Storage |
|---|---|---|
| Free | 10 MB | 500 MB (50 uploads x ~10 MB) |
| Paid | 25 MB | 5 GB (500 uploads x ~10 MB avg) |
| Premium | 50 MB | 25 GB (1000 uploads x ~25 MB avg) |

### 2. Frontend Bucket (`eventalbum-frontend-{env}`)

- **Purpose:** Host React SPA static files (HTML, JS, CSS, images, fonts)
- **Region:** us-east-1
- **Public access:** BLOCKED (served via CloudFront OAC)
- **Encryption:** SSE-S3 (AES-256)
- **Static website hosting:** DISABLED (served via CloudFront OAC, not S3 website endpoint)
- **Versioning:** Disabled
- **Object ownership:** BucketOwnerEnforced

#### Key Structure

```
index.html
assets/
  index-{hash}.js
  index-{hash}.css
  vendor-{hash}.js
favicon.ico
robots.txt
manifest.json
```

## S3 Intelligent-Tiering (Not Used)

Not worth it for this project:
- Minimum object size for tiering: 128 KB
- Monitoring fee: $0.0025/1,000 objects/month
- Our lifecycle rules with tags give us more control and no monitoring fee

## S3 Transfer Acceleration (Not Used)

Not needed — uploads go to us-east-1 via presigned URLs. Guatemala users get reasonable latency. If needed, CloudFront can accelerate uploads via PUT to CloudFront (future optimization).

| Resource | Idle Cost | Notes |
|---|---|---|
| S3 Standard storage | $0.023/GB/month | Only pay for stored data |
